# 快速开始 - 30 秒概览

## 这是什么？

一个关于在 GPU 上加速推荐系统 Beamsearch 的**讨论项目**。

## 核心问题

```
场景：推荐系统混排层 Beamsearch
当前：CPU 20-30ms ❌
目标：GPU < 5ms ✓（5-10 倍提升）

有三类打散规则需要支持：
1. 坑位过滤/强插
2. 窗口 M 出 N
3. 定坑折损
```

## 关键发现

**为什么 GPU 有帮助？**
```
❌ 不能完全并行化所有位置（规则有依赖）
✅ 但每个位置内可以并行检查 2000 个候选

GPU 特别快的操作：
- 广播比较：20x 提升
- 向量求和：10x 提升
- 条件判断：7.5x 提升
```

## 现在需要什么？

你的信息：
1. 打散规则完整列表
2. 候选 item 的完整属性
3. 现有系统的框架（TF/PyTorch/其他）
4. 性能目标（3-5ms 还是 < 1ms？）
